1 Configuring Menus with Presets

The purpose of the preset mechanism is to show only some subset of objects in GUI. The createMenu is reduced, as well as the default-right-click menu and the left-click-on-object menu in the object pool. The set of buttons displayed on selection of an object is reduced, too. No (other) functionality should be reduced.

The subset of objects to be shown is specified in an .rc file, syntax: preset <name> <key>*. It contains object keys and is parsed by interpreter. Only objects referred to by those keys are shown in gui. The key is the className of an object, or if that does not exist (e.g. for .scro), the object name. A preset with name "ALL" exists without an .rc file and shows all objects in GUI.

presets can be set in the console with: thePresetMgr setPreset <name>

The last preset is saved and reloaded on new start of Amira.

In this manager the presets can be chosen without using the console. The actual valid preset is the default, when the manager opens.

To write a preset file with all current modules the Tcl command "writePresetObjects" is introduced.
