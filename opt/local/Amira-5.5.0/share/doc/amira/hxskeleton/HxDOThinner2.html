DOThinner2

  • Module
  • Skeleton Option

Description

DOThinner2 computes a voxel object's skeleton. It uses a variant of the algorithm described in Pudney, C. Distance-ordered homotopic thinning: A skeletonization algorithm for 3D digital images. Computer Vision and Image Understanding 72 (1998), 404.

DOThinner2 only accepts inputs that are loaded to main memory. If you want to avoid loading the data file to main memory, you can use Thinner to process an ExternalData file. Note that loading data to main memory allows more efficient processing.

Connections

Data [required]

The input voxel object (UniformScalarField). DOThinner2 interprets data of type byte as a binary image and data of type short as a distance map.

Locked [optional]

If a UniformScalarField is connected, DOThinner will keep the voxels that are nonzero.

Ports

Len of ends

This port specifies the sensitivity to side branches. A low value tells DOThinner2 to keep short side branches, which might create a lot of spurious branches. A higher value tells DOThinner2 to keep only longer branches.

Max iterations

Nonzero values tell DOThinner2 to break the algorithm after the specified number of iterations. This can be useful for debugging but not for normal processing. The port is hidden by default.
