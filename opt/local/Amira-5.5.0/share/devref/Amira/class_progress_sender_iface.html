Detailed Description

This interface provides the sender part of a progress interface.

The receiver part is provided by

See also:
ProgressReceiverIface. The interface works similar to the old

McProgressInterface but is divided into a sender and a receiver part. The logic of subdividing/undividing a progress should be provided by implementations of this interface. Thus, new receivers can be attached to a sender interface even if the progress is already started. The sender can inform the receiver about its current state by sending the appropriate signals (see for such an implementation). The connection is done using Qt signals and slots. All needed connections can automatically be established by using

ProgressSenderIface::connect().

ProgressSenderIfaceGenImpl provides a ready-to-use implementation of the interface.

void ProgressSenderIface::stopWorking (  )  [inline]

Indicates that time-consuming operation has been completed.

For every call to startWorking() or startWorkingNoStop() there must be a corresponding call to this method. Multiple levels of calls to startWorking() or startWorkingNoStop() are allowed. After stopWorking() is called the previous state of the progress bar and the progress message are restored.

bool ProgressSenderIface::wasSuspended (  )  [inline]

Returns true if the calling thread has been paused inside this method.

This method can be used in a multi-threading context in order for the main thread to suspend and resume a worker thread. The worker thread should frequently call wasSuspended(). Inside this method the thread may wait until some wake-up condition occurs. If this was the case true is returned.

void ProgressSenderIface::startWorking ( int  numSteps,
QString const &  text 
) [inline]

Alternative initialization of the progress bar.

Assuming you know how many steps your operation requires, you can do the following:

  • Initially, call startWorking(numSteps, message)
  • For every step call 
           if (!progressStep()) { stop your work; }
    If you want to do several steps at once specify the number of steps a an argument of progressStep().
  • Finally, call stopWorking().
