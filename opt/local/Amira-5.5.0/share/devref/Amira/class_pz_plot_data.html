Detailed Description

Class to store 2D plot data.

Every plot object which renders data such as PzCurve, PzAnCurve, PzErrorBar and PzLattice has an internal pointer to an object of PzPlotData.

For data which should be rendered as a curve (i.e. 1D data) there must be at least y data. The necessary x data is provided automatically ranging from 0 to n-1. It is possible to change the range with the setMinMax method like this:

   // y is an array of n floats
   myCurve->getTheData()->setValues(n,y);

   // set the data to a range from 10. to 100.
   myCurve->getTheData()->setMinMax(0,10.,100.)

To handle discontinuities in data there are the so called segments which structure the data into separated lines. A segments consists of an array of pairs of indices and length values. Consider the function f(x) = (x-1)/x which is undefined at x==0. If you compute the function in the range [-1 .. 2] with a delta of 0.25 you will get 4 values where x is lower than 0 and 8 values where x is greater than 0. To handle such data in a single curve construct two segments as follows:

   int segments[4];

   segments[0] = 0;  // index of data of first segment
   segments[1] = 4;  // length of first segment
   segments[2] = 4;  // index of data of second segment
   segments[3] = 8;  // length of second segment

   // this call sets new data for 'myCurve' with 2 segments 
   myCurve->getTheData()->setValues(2, segments, x, fx);

To retrieve the data once stored as plot data use the following code snippet within a plot object:

   float *x, *y;
   // we don't want logarithmic values for x and y.
   bool logX = false, logY = false;

   // get values for the first segment
   int n = theData->getValues(&x,&y,logX,logY);
   while (n > 0)
      {
       // ... process or display x and y somehow
       ...

       // get values for the next segment (if any)
       n = theData->getNextValues(&x,&y,logX,logY);
      }
