Detailed Description

Wrapper to use objects of type.

See also:
McProgressInterface with the

ProgressReceiverIface.

This class can be used with new compute modules supporting
See also:
ProgressSenderIface and legacy progress bars (e.g. theWorkArea).
If the connection does not need to negotiate between a worker thread and the main thread,
See also:
ProgressSenderIfaceWrapper can be used instead of this class. It establishes a direct connection between sender and receiver, i.e., no qt connections are used.
Here is a thread-safe example using
See also:
ProgressReceiverIfaceWrapper:
 ProgressSenderIfaceGenImpl progressSender;
 ProgressReceiverIfaceWrapper progressWrapper(theWorkArea);
 progressSender.connect(&progressWrapper);
 
 NewFancyComputeClass fancyCompute;
 fancyCompute.doSomething(&progressSender);

void ProgressReceiverIfaceWrapper::deferredDelete (  ) 

Shedules the object for deletion.

The Object will be delete when control returns to the Qt main thread's event loop (

See also:
QObject::deleteLater()). After the object has been scheduled for deletion it will ignore all calls (signals) except for

ProgressReceiverIface::stopWorking().

This function can be used in situations where it's not feasible to keep a pointer to an instance of this wrapper class. Using this function it will be ensured, that the final
See also:
ProgressReceiverIface::stopWorking() call (signal) won't be lost.
