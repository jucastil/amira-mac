Detailed Description

Dense data field, i.e., a field which can be evaluated everywhere.

A field refers to one or multiple Data objects. In addition, it contains an identifier denoting the interpolation method (MethodId) used to evaluate the field. The interpolation method must match the number and type of data objects. For example, if linear interpolation is used (t_linear), exactly one data object must be referenced. In addition, the data must be defined on nodes and not on edges, faces, or elements.
