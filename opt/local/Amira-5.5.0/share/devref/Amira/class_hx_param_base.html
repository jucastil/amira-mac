enum HxParamBase::Flags

Optional flags.

Most flags are interpreted by Amira's parameter dialog as well as by the Tcl-command interface, but not by the methods of a parameter or a bundle. For example, you may still rename a parameter by calling setName() even if the NO_RENAME flag is set.

Enumerator:
NO_SAVE  Prevents that a parameter or a bundle is written to file.

The flag is interpreted in HxParamBundle::print(). Typically, it will be set for parameters that are used internally, e.g. loadCmd or saveInfo.

NO_COPY  Prevents that a parameter or a bundle is copied into another bundle.

This may happen if the following methods of class HxParamBundle are called: the copy constructor, the assignment operator, or copy(const HxParamBundle&). The flag is used to prevent that parameters like loadCmd are copied to a data object newly created by a computational module.

NO_DELETE  Prevents that a parameter or a bundle is removed in Amira's parameter editor.

This flag is used to prevent that the users remove materials from a LabelField or a TetraGrid. A bundle containing a parameter with the NO_DELETE flag set may still be removed from its parent bundle. You have to set the NO_DELETE flag for all parent bundles if you want to prevent this.

NO_INSERT_BUNDLE  Prevents that a new bundle can be inserted into an existing bundle.
NO_INSERT_PARAM  Prevents that a new parameter can be inserted into an existing bundle.
NO_RENAME  Prevents changing the name of a parameter or a bundle.
NO_SETVALUE  Prevents that the value of a parameter is changed by one of HxParameter's or HxParamBundle's set methods.

If set for a bundle the bundle may ot be replaced by another one with equal name using insert().

enum HxParamBase::NewMask

Elements of the isNew()-mask.

Enumerator:
NEW_NAME  Set if the name of a parameter or a bundle is changed using setName().
NEW_VALUE  Set if the value of a parameter is changed or if an element of a bundle is inserted or removed.
NEW_NAME_CHILD  Set in all parent bundles if a name of a parameter or a bundle is changed.
NEW_VALUE_CHILD  Set in all parent bundles is the value of a parameter is changed or if an element of a bundle is inserted or removed.
MODIFIED  Set if anything changed.

Indicates that a bundle needs to be saved.

HxParamEditCheck* HxParamBase::editCheck (  ) 

Returns pointer to edit checker.

If this class does not itself provides an edit checker the checker of the class's parent bundle is returned. If there is no parent bundle a pointer to a default checker is returned. The default checker interpretes the NO_RENAME, NO_DELETE, NO_INSERT, and NO_SETVALUE flags of a parameter.
