Detailed Description

This class provides an interface to user-defined file formats.

File formats are registered by means of an Amira Tcl-command. Both, loading and saving of Amira objects is supported.

In order to register a file format use something like this:

    dataFile -name GIF-option gif \
        -ext .gif -load readGIF \
        -type HxRGBImage -save saveGIF \
        -dso libimio.so

See the usersguide for a list of Fileformats already supported by Amira.

int HxFileFormat::load ( int  n,
const char *const *  files,
bool  loadAsync = true 
)

Try to load given files.

The files are loaded synchronously if either loadAsync is set to false or a network is currently loading. Otherwise all files are loaded asynchronously. When loading synchronously the number of successfully read files is returned, otherwise the number of scheduled files. If a file could not be read the method returns without trying to load further files.

char **  argv 
) [static]

Parses arguments and returns a new instance of HxFormat.

The following options are interpreted:

  • -label Label
  • -option Option
  • -header HeaderRegEx
  • -file FilenameRegEx
  • -ext Suffix1[,Suffix2,...]
  • -load ReadMethod
  • -type SaveType1[,SaveType2,...]
  • -save SaveMethod
  • -check Tcl-expression
  • -multisave SaveMethod
  • -dso SharedObjectName
  • -loadInOwnThread <false|true|multi>
Note, that argv[0] should already point to the first option.

static void HxFileFormat::enumerate ( McString &  result,
const char *  pattern,
int  n 
) [static]

Generate an enumerated file name.

pattern is searched for a sequence of #-characters. If such a sequence is found it is replaced by the number n printed with leading zeros. If no such pattern is found the number is inserted before the filename suffix using at least four digits. If no suffix exists it is appended to the filename.
