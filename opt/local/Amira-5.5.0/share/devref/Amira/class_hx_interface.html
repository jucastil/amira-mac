Detailed Description

Provides a common API for objects not related by inheritance.

Interfaces can be secondary base classes or members of a data object or a module. The first approach is not recommended since multiple inheritance sometimes may cause undesired effects. In any case, interfaces must be registered with the object they belong to, i.e., an owner object must be defined (see constructor or setOwner).

You may configure a HxConnection in such a way that it can be connected to any object providing an interface of a certain type. The object itself must not necessarily derived from this type. Likewise, you may define a save function (see HxFileFormat) for an interface. Instead of a pointer to an object derived from HxData the save function then is called with a pointer to the interface.

Interfaces also provide a Tcl command interface (parse). Commands defined in an interface override commands defined in parse method of the owner object itself.
