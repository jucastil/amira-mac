Member Data Documentation

bool HxPreferences::drawComputeIndicator

If true, execution of an object's compute method is indicated graphically.

The rectangular connection area of the object's icon is drawn in red during execution.

bool HxPreferences::autoSelectNewModules

If true, new modules are selected automatically after they have been created via an object's popup menu.

bool HxPreferences::deselectPreviouslySelectedModules

If true, and if autoSelectNewModules is set selected objects are deselected before a new object is selected.

bool HxPreferences::drawViewerTogglesOnIcons

If true, the object's viewer toggles are drawn on the icon.

If not set, the viewer toggles are only drawn on the object's header bar. In this case the object must be selected in order to modify the viewer mask.

bool HxPreferences::twoPassFiring

If true, a 2-pass firing mechanism is used.

This mechanism ensures that in complex networks downstream modules will only be fired once.

bool HxPreferences::includeUnusedDataObjects

If true, all objects including hidden once are saved in networks.

Otherwise hidden data objects with no down stream objects attached to them (such as colormaps) will not be saved. Default is off.

bool HxPreferences::overwriteInAutoSave

If true, old files are overwritten during auto save.

This applies to data objects which need to be saved to file before a network script / can be generated.

bool HxPreferences::doNotShowAnyNews

bool HxPreferences::includeWindowSizeAndPosition

If true, the size and the position of Amira viewer windows will be included in network files.

This results in scripts which are not fully portable because the screen resolution as well as the size of the window borders is not the same of all systems. On default, this option is off.

bool HxPreferences::onExitCheckDataSave

If true, Amira checks if there are unsaved data objects before quiting.

If so a message box listing these data objects is displayed. The user then can save the objects, quit Amira anyway, or cancel the dialog and return to Amira.

bool HxPreferences::onExitCheckNetworkSave

If true, Amira checks if the network needs to be saved before quitting, The conditions indicating that the network need to be saved can be controlled in more detail using the flags below.

bool HxPreferences::networkDirtyAddRemove

If true, the network is marked as dirty when objects are added to or removed from the object pool.

bool HxPreferences::networkDirtyXform

bool HxPreferences::networkDirtyLabels

bool HxPreferences::axisNamingIsMedical

The medical naming is the default.
