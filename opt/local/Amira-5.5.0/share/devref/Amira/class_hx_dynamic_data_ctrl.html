Detailed Description

Base class for modules managing dynamic (time-dependent) data.

Currently, in Amira there are no or not many classes which directly represent time-dependent data. As a workaround ordinary static data objects can be created. These objects can be updated in order to represent other time steps.

To update static data objects it is most convenient to design a special-purpose compute module. The class HxDynamicDataCtrl is intended to be the base class of such modules. It provides a time port. Whenever the time changes one of the virtual methods newTimeStep respectively newPhysicalTime is called. Derived classes should overload these method.
