Detailed Description

template<class T>
class McDArray< T >

This class provides a dynamic array of movable C++ objects.

C++ constructors and destructors are called correctly, but when reallocating memory the objects may be moved using memmove(). Therefore you must not store the address of an individual element elsewhere. This class requires the template parameter class to have a public default constructor.

The elements of an instance of McDArray can be accessed just like ordinary C++ array elements by means of the index operator[]. Safety is provided by

  • automatic memory management in constructor and destructor preventing memory leaks
  • checking of array bounds when accessing elements with the indexing operator[] (only when compiled without -DNDEBUG).
In order to support 64-bit architectures the type mclong is used for all index variables. mclong (defined in McPrimType.h) is equivalent to ssize_t which is either 32-bit or 64-bit, depending on which architecture is being used.

McDArrays may easily be extended by inserting or appending elements to the McDArray or shrunken by removing elements. The method resize(mclong n) resets the size of the array to n, thereby possibly appending elements or removing ones. Note that for built-in types the default constructor is not called on new elements for efficiency.

McDArrays may be used as arguments for standard C functions requiring pointers through a cast operator.

Internally, a McDArray object allocates a block of memory that fits up to max() elements, only size() of them are used. This makes extension and shrinking methods perform better.

See also:
McSmallArray
For template functions for searching and comparisons see index(), rindex() and mcStandardCompare().

template<class T>
mclong McDArray< T >::remax ( mclong  newMax = -1,
mclong  newSize = -1 
) [inline]

Reset size of internal buffer.

The value of max() is reset to newMax thereby setting size() to newSize. However, if newSize has a value < 0 (as the default argument does) size() remains unchanged and max() is set to MAX(size(), newMax). Hence, calling remax() without the default arguments, will reduce the memory consumption to a minimum. In no instance max() will be set to a value less than 0 (even if specified).

Returns:
The number that max() is reduced.
