Detailed Description

The abstract base class for field evaluators.

This class allows you to evaluate any field derived from HxField3 at an arbitrary point in space. In contrast to the methods HxField3::createLocation() and HxField3::eval() field evaluators allow you to choose between different interpolation methods. HxField3::eval() always uses a standard interpolation scheme, e.g., linear interpolation for fields on tetrahedral grids or trilinear interpolation for fields with regular coordinates. In addition, the HxFieldEvaluator interface provides nearest-neighbor interpolation as well as high-quality Lanczos interpolation. However, these methods may not be provided for all types of data fields.

For example, if you prefer to use nearest-neighbour interpolation but don't mind to use standard interpolation if this is not available for a given field, you should use the following code: 

        HxFieldEvaluator::Method preferred = HxFieldEvaluator::EVAL_NN;
        HxFieldEvaluator* eval = HxFieldEvaluator::createEval(field, preferred);
        HxLocation3* loc = eval->createLocation();

        loc->move(x, y, z);
        eval->eval(loc, &result);

        delete loc;
        delete eval;

This will always work. If the preferred interpolation method is not available, HxFieldEvaluator::createEval() will return an instance of HxEvalStandard, which applies the standard interpolation method of the field.

If you want to insist on nearest-neighbor interpolation you should call HxEvalNN::createEval() and check the result of this method: 

        HxFieldEvaluator* eval = HxEvalNN::createEval(field);
        if (eval) {
            HxLocation3* loc = eval->createLocation();

            loc->move(x, y, z);
            eval->eval(loc, &result);

            delete loc;
            delete eval;
        }
        else {
            // perform some error handling here
        }

static HxFieldEvaluator* HxFieldEvaluator::createEval ( HxField3 *  field,
Method  preferredMethod = EVAL_NN 
) [static]

Factory method.

Returns an object that knows how to deal with the given field. preferredMethod denotes the preferred interpolation method. If this method is not available for the given field, a standard evaluator will be returned. Thus the method never returns null. The returnd evaluator should be deleted by the user.

virtual void HxFieldEvaluator::eval ( HxLocation3 *  loc,
float *  result 
) [pure virtual]

Evaluates the field.

loc describes the location where the field should be evaluated. The location can be specified using HxLocation3::set() or HxLocation3::move(). The interpolated field values are always casted to float and are stored at result. result must point to an array that is large enough to store as many values as indicated by HxField3::nDataVar().

Implemented in HxEvalStandard.

Evaluates the field.

This methods behaves exactly like HxFieldEvaluator::eval() but it does not cast the interpolated field values to float. Instead the result is returned in the same data type as indicated by HxField3::primType().
