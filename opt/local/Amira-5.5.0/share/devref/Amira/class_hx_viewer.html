Detailed Description

Amira's customized viewer class.

This class encapsulated a customized Open Inventor examiner viewer (QxViewer, derived from SoQtExaminerViewer). The class allows amiraDev users to access the viewer without needing to install any Qt headers. It is simply a wrapper for QxViewer.

void HxViewer::setBackgroundImage ( const int  dims[3],
const unsigned char *  data 
)

Sets a background image.

The width of the image, its height, as well as the number of color channels must be stored in dims[0], dims[1], and dims[2], respectively. The actual image data must be stored in data. The image pixels must be stored in contiguous order without any padding between the lines. Only 8-bit data is supported. The number of color channels must be either 1 (greyscale data), 3 (rgb data), or 4 (rgba data). The image data is copied into an internal buffer.

Reads a stereo background image from one or two fles.

If two file names are specified, it is assumed that these files represent the left eye view and the right eye view, respectively. void HxViewer::snapshot ( int  outSize[3],
unsigned char *&  outData,
int  inNumTilesX,
int  inNumTilesY,
int  inMakeAlpha,
int  inBytesPerPixel,
CameraMode  inCamMode 
)

Creates a snapshot from the viewer and returns the image data in outData.

The memory must be freed by user. The images dimensions (width, height, number of color channels) are delivered in outSize. If inNumTilesX or inNumTilesY is bigger than one, tiled rendering is applied. When the snapshot is taken with inCamMode set to ACTIVE_STEREO, the resulting images are twice as wide as the window. In the left half of the image the left eye snapshot is stored and in the right half the right eye snapshot. If inMakeAlpha is nonzero the resulting image has an alpha channel which blends out the background.
