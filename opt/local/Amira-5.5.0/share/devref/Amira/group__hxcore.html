Required for compatibility with Amira 2.x.

In Amira 3.0 the class HxPort3DPoint has been removed because it merely provided a subset of the functionality of HxPort3DPointList. With this define old code should compile as is with one possible expection. In the old class HxPort3DPoint the method getValue(int n) returned a float, namely the n-th component of the 3D point. In HxPort3DPointList the method getValue(int n) returns an SbVec3f, namely the n-th point in the list. In order to fix this you have to replace getValue(k) by getValue()[k]. If getValue() was used without arguments (which should be the standard case) nothing has to be done.
